# Hermes 全系统中断根因扫描报告
# 时间: 2026-05-05 格林主人指令
# 找到全部致命bug，逐条修复

## BUG #1 — 采集器: collect_platform() 线程超时15秒，但所有长耗时平台都被静默吞掉
- 位置: unified_collector_v5.py:793 — `t.join(timeout=15)`
- 根因: 线程15秒超时后返回空结果，但不打印任何警告
- 影响: hackernews/huggingface/reddit 永远返回0条（因为它们需要≥15秒）
- 修复: 把超时提到30秒，并记录超时日志

## BUG #2 — 采集器: fetch() 裸except通吃所有异常，永不输出错误
- 位置: unified_collector_v5.py:138 — `except: return ""`
- 根因: 网络错误、SSL错误、DNS错误全被吞掉，返回空字符串
- 影响: 用户永远不知道采集器为什么没采集到某个平台
- 修复: 打印具体异常

## BUG #3 — guardian.py: 没有"如果某项任务失败，自动恢复"的闭环
- 位置: guardian.py:36-50 run()函数
- 根因: 重试2次后返回空字符串，调用方不检查返回值
- 影响: 如果采集失败，整个cycle静默完成但实际上没采集到任何数据
- 修复: 检查run()返回值，失败时重新调度

## BUG #4 — guardian.py: push动作不检查推送候选是否过时
- 位置: guardian.py:347-368 push部分
- 根因: 如果候选文件是2小时前的旧数据，还用它推送
- 影响: 推送的内容可能过时
- 修复: 检查候选文件生成时间，超过2小时先跑cycle再推

## BUG #5 — 守护神启动时没有修复旧cron的锁定文件
- 位置: cron/.tick.lock 文件在上一轮完成后没有被清理
- 根因: cron调度引擎没运行的原因
- 影响: 下次调度可能被锁阻止
- 修复: heal时自动清理锁定文件

## BUG #6 — 采集器重复数据累计
- 位置: unified_collector_v5.py 各collect函数
- 根因: 同一个平台每次采集到相同热门数据，大量重复入库
- 影响: 清洗管道处理3580条数据时3369条是重复
- 修复: 采集时先检查最近N小时是否有相同标题

## BUG #7 — 没有检查磁盘空间
- 根因: 磁盘满了会导致DB写入失败、脚本崩溃
- 影响: 整个系统停摆
- 修复: heal时检查磁盘空间

## BUG #8 — 守护神日志没有轮转，可能撑爆磁盘
- 根因: 每天一个log文件，无限append
- 影响: 长期运行log文件越来越大
- 修复: 日志超过10MB自动截断
